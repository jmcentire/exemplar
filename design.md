Build a **governed code review service** — a CLI + MCP tool that accepts pull request diffs, routes them through multi-stage analysis (security, correctness, style, architectur
  - [ ] **Assessment Merger & Trust Scoring** (`assessor`)
    Merges multiple reviewer Assessments into a single ReviewReport using Arbiter trust-weighted scoring. Conflict resolution: security BLOCK overrides style PASS with explanation; configurable priority ordering. Per-finding confidence scores derived from reviewer trust weight × finding confidence. Deduplicates overlapping findings across reviewers. Produces final ReviewDecision (block/warn/pass) based on configurable thresholds. Emits assessment.merged Chronicler event.
  - [ ] **Baton Circuit Router** (`circuit`)
    Async circuit routing engine. Takes classified DiffHunks and routes them through configured review stages per CircuitConfig. Supports parallel execution of independent stages via asyncio.gather, sequential execution where ordering matters. Implements circuit-breaker pattern: a failing/timing-out reviewer produces a degraded Assessment (recorded as partial result) rather than crashing the pipeline. Emits Chronicler events at stage boundaries (stage.started, stage.complete). Returns ordered list of Assessments from all stages. Configurable timeout per stage.
  - [ ] **CLI Entry Point** (`cli`)
    argparse-based CLI with subcommands: `exemplar review <diff-file> [--config path] [--format json|md|github]` (full pipeline: intake → circuit → assess → report), `exemplar trust` (display current reviewer trust scores), `exemplar history [--query term]` (Kindex query of past reviews), `exemplar adopt` (Cartographer: scan cwd for .eslintrc, .pylintrc, .flake8, etc. and report findings with migration suggestions). Thin transport layer over shared core modules. Async event loop management. Exit codes: 0=pass, 1=warn, 2=block, 3=error. Emits review.started Chronicler event.
  - [ ] **Configuration System** (`config`)
    YAML + Pydantic config loading with fail-fast validation. Defines ExemplarConfig (top-level), CircuitConfig (stage ordering, parallelism, thresholds), ReviewerConfig (per-reviewer: enabled, trust weight default, Agent-Safe scope patterns), LedgerConfig (field classification rules for secrets/PII/internal APIs), ApprenticeConfig (shadow/canary/primary mode, storage path). Provides default config that works out-of-the-box. Constrain definitions are expressed here: allowed severity levels, excluded file patterns, scope boundaries per reviewer. Config path resolution: explicit flag > env var > .exemplar/config.yaml > built-in defaults.
  - [ ] **Governance Primitives** (`governance`)
    Inline implementations of governance stack patterns used across modules. Includes: TesseraSealer (SHA-256 hash-chain seal creation and verification over canonical JSON), ChronicleEmitter (emit structured events to JSON-lines log file, queryable), AgentSafeEnforcer (filter hunks by PolicyToken scope, log violations), SignetManager (create/verify reviewer credentials), ArbiterScorer (weighted trust score calculation and conflict resolution), LedgerClassifier (classify diff content against field rules for secrets/PII/internal APIs), StigmergyTracker (record and query recurring pattern signals to JSON storage), KindexStore (persistent JSON-file key-value store for review history and codebase context). Each is a focused class with a clean Protocol-based interface, fire-and-forget for optional integrations.
  - [ ] **Diff Intake & Classification** (`intake`)
    Async module that parses unified diff text into DiffHunk records. Extracts file paths, changed line numbers, context lines, and hunk metadata. Applies Ledger field classification rules to flag hunks containing potential secrets, PII, or internal API references. Output: list of classified DiffHunks ready for circuit routing. Handles malformed diffs gracefully with clear error messages. Supports reading from file path or stdin.
  - [ ] **Apprentice Learning Module** (`learner`)
    Shadow mode implementation: accepts AI review output and human review decisions (as JSON input), compares them, records acceptance/dismissal rates per reviewer and per finding type. Stores learning state in JSON files (.exemplar/learner/). Feeds acceptance patterns back as trust score adjustments. Phase progression tracking (shadow → canary → primary) based on configurable accuracy thresholds. Stigmergy integration: reports recurring patterns (e.g., 'SecurityReviewer false-positive rate on test files is 40%') to pattern store.
  - [ ] **MCP Server** (`mcp_server`)
    MCP tool server over stdio transport exposing three tools: exemplar_review (accepts diff text or file path, config overrides, format preference; returns formatted ReviewReport), exemplar_trust (returns current trust scores as structured data), exemplar_history (accepts query parameters, returns matching past reviews from Kindex). Uses same core pipeline as CLI. Implements MCP protocol message handling (JSON-RPC over stdio). Tool schemas generated from Pydantic models. Async request handling.
  - [C] **Report Formatter & Sealer** (`reporter`)
    Formats ReviewReport into three output formats: JSON (validates against schema), Markdown (human-readable with severity badges), GitHub-compatible PR comment (collapsible sections per stage). Deterministic output: injectable timestamps, sorted keys, canonical serialization. Applies Tessera seal to final report (hash-chain proof over content). Includes verify function that confirms seal integrity and detects post-seal modification. Emits report.sealed Chronicler event.
  - [C] **Reviewer Implementations** (`reviewers`)
    ReviewerProtocol (runtime_checkable Protocol with async review method accepting list of DiffHunks and returning Assessment). Four implementations: SecurityReviewer (regex/pattern detection for hardcoded secrets, SQL injection, path traversal, OWASP top patterns), CorrectnessReviewer (AST-based analysis for None dereference patterns, off-by-one in range/slice, unchecked return values, logic errors), StyleReviewer (convention violations: naming, line length, import ordering, docstring presence), ArchitectureReviewer (coupling detection via import analysis, layer violation checks, circular dependency hints). Each reviewer is scoped by Agent-Safe PolicyToken — receives only permitted hunks. Each produces typed findings with Severity and Confidence. Heuristic-quality, not production static analysis.
  - [C] **Data Models & Schemas** (`schemas`)
    Frozen Pydantic v2 models and StrEnums forming the shared data contract for the entire system. Includes: DiffHunk (parsed diff unit), ReviewRequest (intake output), Assessment (single reviewer output with findings), ReviewReport (merged final output), ReviewerCredential (Signet identity), TrustScore (Arbiter weight), PolicyToken (Agent-Safe scope), ChroniclerEvent (event log entry), TesseraSeal (hash-chain proof). StrEnums: Severity (critical/high/medium/low/info), Confidence (high/medium/low), ReviewStage (security/correctness/style/architecture), ReviewDecision (block/warn/pass). All models frozen=True, full JSON round-trip support, canonical serialization method for deterministic hashing.
